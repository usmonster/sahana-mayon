Sahana Agasti Documentation

What Is Agasti?

Agasti is the PHP based project of the Sahana Software Foundation. Based a long-term preparedness for disaster management, Agasti aims to provide scalability; whether for major cities or micro event response.

With the formation of the Agasti Project Management Committee, the PHP branch of Sahana has a new burst of activity and momentum as well as the history of Sahana behind it. There are currently three versions of Agasti with active development: Krakatoa (formerly Sahana 0.6.3), Vesuvius, and the newest version Mayon.

Each version is actively seeking contributors including application developers, documentation writers, translators, managers, QA specialists, and emergency response specialists interested in helping to refine the current versions, and push ahead with creation of the newest versions. For more information about how you can help visit the Agasti wiki at http://wiki.sahanafoundation.org/doku.php/agasti:start.

Agasti's Mission

The mission of Sahana Agasti is to facilitate resource management and communication throughout all phases of emergency management by providing a turnkey open-source application that is reliable, scalable, extensible, and usable both by volunteers and professionals, and which supports the primary goal of the Sahana Software Foundation: alleviating human suffering and saving lives.

System Requirements

Software

The following technology stack is assumed to be present and running on the host machine:
* Ubuntu >= 9.10 LTS (most current 10.04 LTS strongly recommended)
* Apache HTTP Server >= 2.2.14 (most current 2.2 preferred)
* MySQL >= 5.1.41 (most current 5.1 preferred)
* PHP >= 5.2.12 (most current 5.3 strongly recommended)

Hardware

The recommended hardware resources include:
* Intel Pentium or AMD Athlon CPU
* minimum of 50 MB available disk space
* minimum 512 MB RAM

Known Issues

* IMPORTANT GENERAL NOTE: this is the initial trunk foundation for the Mayon release. It is not intended for production use and the devlopment team is aware multiple modules may be missing and functionality is limited.

* After initially filling in the install.php form it may take up to 5 minutes to load and redirect to the login screen.

* Many navigation links point to home for modules that will be implemented in future releases.

* Pre-loaded fixtures for staff are not searchable; this is because fixture data is not currently indexed for searching when populating the schema through the install process. Any new data entered into the system, however, will be searchable.

* The formatting on some pages may seem "off." Page elements may wrap or disappear based on your browser window size or screen resolution. As a temporary workaround, you may need to maximize your browser window or increase your screen resolution.

* Caching solutions are not included with this version of trunk.

If you think you've found a new issue or bug, please refer to the "Support" section above for information on how to contact our volunteer community.
